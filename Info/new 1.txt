В данной мастерской достаточно предъявить то, что нужно сделать, указать количество таких изделий и дать ресурсы.
В мастерской имеется книга рецептов изделий. 
	Книга конечна.
	В книге отсутствуют непосредственно  рецепты. Это такой рецепт, в ингредиентах которого есть ссылка на этот же рецепт, т.е. (x, A) и x \have A.
	В книге существуют посредственно самоссылающиеся рецепты. Это такой перечень рецептов, при использовании которого в операции разворачивания цепи для изготовления изделия x, возникает ингредиент x. 
		К примеру, дана книга из [(a, [b]); (b, [a])] рецептов.
		Работа операции expand будет следующей:
			для изготовления a необходимы:
				ингредиент b, для изготовления которого необходимы:
					ингредиент a, но если попытаться сделать изделие a то цикл станет бесконечным.
		
		книга из [(a, [b]); (b, [c]); (c, [b])] рецептов.
		Работа операции expand будет следующей:
			для изготовления a необходимы:
				ингредиент b, для изготовления которого необходимы:
					ингредиент c, для изготовления которого необходимы:
						ингредиент b, но если попытаться сделать изделие b то цикл станет бесконечным.
		
		функцию можно переделать так, чтобы она возвращала следующее:
		книга из [(a, [b]); (b, [a])] рецептов
			для изготовления a необходимы:
				ингредиент b
		книга из [(a, [b]); (b, [c]); (c, [b])] рецептов.
		Работа операции expand будет следующей:
			для изготовления a необходимы:
				ингредиент b, для изготовления которого необходимы:
					ингредиент c
		книга из [(a, [b]); (b, [c]); (c, [a])] рецептов.
		Работа операции expand будет следующей:
			для изготовления a необходимы:
				ингредиент b, для изготовления которого необходимы:
					ингредиент c (поскольку c можно сделать только используя ингредиент a, а в данной цепочке делается элемент a -> то такой элемент изготовить не возможно)
		

	Разворачивание цепи рецептов для изготовления изделия x - функция expand, составляющая следующую цепочку высказываний:
		к примеру книга рецептов состоит из [(a, [b; c]); (b, [c; d]); (c, [e; f]);], необходимо изготовить изделие a
		Результат функции выглядит так:
		для изготовления a необходимы:
			ингредиент b, для изготовления которого необходимы:
				ингредиент с, для изготовления которого необходимы:
					ингредиент e, который отсутствует в книге рецептов
					ингредиент f, который отсутствует в книге рецептов
				ингредиент d, который отсутствует в книге рецептов
			ингредиент с, для изготовления которого необходимы:
					ингредиент e, который отсутствует в книге рецептов
					ингредиент f, который отсутствует в книге рецептов
	
	Существуют рецепты, которые могут дать больше одного изделия.
	Поскольку есть рецепты, дающие в результате больше одного изделия, в expand может возникнуть дополнительные изделия.
		к примеру книга рецептов состоит из [(a, 2, [b; c; g]); (b, 3, [c; d]); (c, 1, [e; f]); (d, 2, [e; f]); (g, 2, [d])] (имяИзделия * количествоПолучаемогоИзделия * списокИнгридиентов), необходимо изготовить одно изделие a
		Результат функции выглядит так:
		рецепт изготовления a дает 2 изделия. Нам нужен 1, записываем в остаток ost = [a, 2-1]. Для изготовления рецепта необходимо:
			рецепт изготовления b дает 3 изделия. Нам нужен 1, записываем в остаток ost2 = (b, 2). Для изготовления рецепта необходимо:
				рецепт изготовления c дает 1 изделия. Нам нужен 1, остатка нет. Для изготовления рецепта необходимо:
					e
					f
				рецепт изготовления d дает 2 изделия. Нам нужен 1, записываем в остаток ost3 = (d, 1). Для изготовления рецепта необходимо:
					e
					f
			общий остаток ost4 = ost2 + ost3 = [b, 2; d, 1]
			рецепт изготовления c дает 1 изделия. Нам нужен 1, остатка нет. Для изготовления рецепта необходимо:
					e
					f
			рецепт изготовления g дает 2 изделия. Нам нужен 1, записываем в остаток ost5 = (g, 1). Для изготовления рецепта необходимо:
				d	поскольку в ost4 есть изделие d, используем его из остатка. В результате ost6 = [b,2; d, 1-1] = [b,2]
			общий остаток ost7 = ost5 + ost6 = [g, 1; b, 2;]
	общий остаток ost7 = [g, 1; b, 2;]

Ресурсы можно дать любые и в любом количестве. Мастерская работает добросовестно: берет ровно столько ресурсов для изделия, сколько написано в рецепте и все остальные ресурсы возвращает.
